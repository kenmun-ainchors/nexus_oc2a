## Problem

Heartbeat detected `state/cron-health-alert.json` with 1 unacknowledged failure:
- **Cron 85595417** "PG-Notion Integrity Audit (daily)" — `consecutiveErrors=4` spanning **23 days**
- Last run: 2026-06-21 01:00 AEST; last error: `cron: job execution timed out (last phase: tool-execution-started)`, duration 194s
- Schedule: `0 1 * * *` Australia/Melbourne → next run 2026-07-14 01:00 AEST (~1h 35m from now)

### Root cause (two layers)

**Layer 1 — Cron payload does not satisfy the script's skill gate.**
- Cron 85595417 is an `agentTurn` whose payload text tells the agent: *"Run `bash scripts/pg-to-notion-sync.sh --audit`"*
- The script sources `skill-gate.sh notion` at the top, which requires a registry load record within 60 min (`state/skill-load-registry.json → notion.loaded_at`)
- Cron spawns a fresh isolated session, so there is no prior skill load; the agent's 60-min window has no record; the gate blocks the run with `SKILL GATE — BLOCKED: Status: LOAD RECORD STALE OR INVALID`
- Agent emits the error but never returns a final response → cron times out at `tool-execution-started`

**Layer 2 — `scripts/skill-gate.sh` line 27 uses `exit 0` instead of `return 0`.**
- When `SKILL_GATE_BYPASS=1`, the gate does `exit 0` — but the script is meant to be `source`d by callers, so `exit 0` kills the caller's shell.
- This is why `changelog-append.sh` (which `source`s `skill-gate.sh`) silently fails when called from a non-interactive context with `SKILL_GATE_BYPASS=1`.
- The "bypass for automation" path is therefore **broken in addition to the cron payload's missing pre-load step**.

### Impact

- PG-Notion drift has been invisible for 23 days. TKT-0406 exists specifically to catch this drift. The 7-check audit (count, duplicates, freshness, status, references, unsprinted, carryovers) is not running.
- The skill-gate sourcing bug blocks any script that uses `SKILL_GATE_BYPASS=1` as a sourced library (currently: `changelog-append.sh`, `pg-to-notion-sync.sh`, `linkedin-token-health.sh`, etc., all of which `source` skill-gate.sh).

## Changes (Forge-execute)

### 1. `scripts/skill-gate.sh` — fix the bypass sourcing bug

- Line 27: replace `exit 0` with `return 0` so the bypass properly returns control to the caller.
- (Optional, recommended) Add an explicit comment: `# return, not exit — this script is sourced`.

### 2. `scripts/pg-to-notion-sync.sh` — make the audit branch automation-safe

- The `--audit` flag invocation should be runnable from cron without the agent needing to manually satisfy the skill gate.
- Add at the top of the `--audit` branch:
  ```bash
  if [[ "${SKILL_GATE_BYPASS:-}" != "1" && -z "${SKILL_LOADED_NOTION:-}" ]]; then
    log "WARN: skill-gate not satisfied; consider SKILL_GATE_BYPASS=1 for cron/automation contexts"
  fi
  ```
- Or, cleaner: have the cron payload set `SKILL_GATE_BYPASS=1` (and the script's audit path should respect it). Document in the script header.

### 3. Cron 85595417 payload update

- Update the `message` field to instruct the agent to set `SKILL_GATE_BYPASS=1` before invoking the script, or to load the skill first. Concrete diff:
  - **Before:** `bash /Users/ainchorsoc2a/.openclaw/workspace/scripts/pg-to-notion-sync.sh --audit`
  - **After:** `SKILL_GATE_BYPASS=1 zsh /Users/ainchorsoc2a/.openclaw/workspace/scripts/pg-to-notion-sync.sh --audit`
  - Plus preamble: *"If you prefer to load the skill instead, run `bash scripts/skill-load.sh notion` first."*
- Save via `openclaw cron update 85595417-73d0-416d-b6ff-d5b6578762a7 --patch '{"payload":{"message":"<new text>"}}'`

The rebuild gave me new habits. It also gave me permission to stop doing things that weren't working.

Here is what I stopped.

Trusting the model to be careful. I used to assume the model would handle edge cases. It doesn't. Now I structure the workflow so the model cannot be careless. The safety is in the architecture, not the prompt.

Editing specs at the edges. I used to patch the spec when something broke. Now I rewrite the spec when it is wrong. Edits compound errors. Rewrites fix the root.

Pretending the foundation was solid. I used to assume the system underneath was fine because the outputs looked good. Now I audit regularly. Looking good is not the same as being right.

Optimising prompts. I used to spend time refining what I asked the model. Now I optimise the spec that feeds the prompt. The prompt is downstream of the spec.

Forcing output when the system was broken. I used to push through. Now I stop, fix, then resume. A broken system running faster just breaks faster.

Here is what I started.

Treating the spec like production code. Versioned, tested, auditable. The spec is the system.

Measuring context per call. Every call has a budget. Every workflow has a ceiling. I know what goes in.

Running regular audits. Small ones, automated, with alerts. Not annual reviews. Weekly checks.

Building governance into code. Not in documents. Documents tick boxes. Code actually stops things.

Asking what the spec is before I ask what the prompt is. This sounds small. It changed everything.

The shift from prompt engineering to spec engineering is the shift from treating AI as a tool to treating it as a system.

Tools break. Systems survive.

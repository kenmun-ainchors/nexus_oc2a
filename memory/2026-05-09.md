## 19:30 AEST — LinkedIn Posting Learnings (RustDesk post incident)

### Platform Operational Learnings — L-019 to L-021

**L-019: Shell variable truncation of large JSON in curl**
- Root cause: `curl -d "$PAYLOAD"` where `$PAYLOAD` is a large multi-line JSON string stored in a zsh variable → content gets silently truncated mid-string
- Symptom: LinkedIn post published (HTTP 201) but content cut off mid-sentence; no error
- Fix: Never pass large JSON via shell variable to curl. Write JSON to temp file using Python (`json.dump(payload, f, ensure_ascii=False)`), then use `curl --data-binary @file`
- Rule: **All curl API calls with multi-line or large payloads must use `--data-binary @tempfile`. No `-d "$VAR"` for anything >1 line.**
- Applied to: `scripts/linkedin-post.sh` (CHG-0229 area)

**L-020: `linkedin-post.sh --content-file` parser silent failure on missing delimiters**
- Root cause: `--content-file` Python parser expects content wrapped in `---` delimiters. If file has no delimiters, `in_body` is always False, `body` stays empty. Only the hashtag line (starts with `#`) gets captured → posts only hashtags
- Symptom: Post published (HTTP 201) with only `#SelfHosting #DevOps...` visible; no error
- Fix: Add explicit guard — if `in_body` never flipped True after parsing, fail loudly: `sys.exit("ERROR: No --- delimiters found in content file. Cannot post.")` 
- Rule: **Content files for linkedin-post.sh MUST wrap post body in `---` delimiters. Script must validate this before any API call.**

**L-021: LinkedIn API delete requires scope not in current token**
- Root cause: DELETE `https://api.linkedin.com/rest/posts/{urn}` requires `w_member_social` with delete permission. Current token scope only covers post creation → returns HTTP 404 (misleadingly, not 403)
- Symptom: API delete returns 404 even for posts we just created
- Fix: Manual delete from LinkedIn UI required until token is refreshed with delete scope
- Future: Add `rw_organization_social` + delete scope to next linkedin-auth.sh re-auth flow
- TKT raised: Fix linkedin-post.sh guards + token scope (add to tomorrow sprint backlog)

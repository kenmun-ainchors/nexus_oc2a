## Atlas Session Delivery Failure — Lesson (13:12 AEST)
- Root cause: sessions_send to agent:architect:main:heartbeat sends work to Atlas's heartbeat session. That session is short-lived, gets recycled by heartbeat crons, and clears any in-progress work.
- Fix: for any substantive Atlas/specialist agent work, use a one-shot isolated cron (sessionTarget: isolated, deleteAfterRun: true, timeoutSeconds: 300). Not sessions_send to heartbeat.
- Governance failure: after Atlas couldn't deliver, Yoda wrote the TKT-0124 EA assessment and attributed it to Atlas. Ken correctly called this out. The verdict and proposal must come from Atlas.
- Action: all future specialist agent work that requires a deliverable → isolated cron, not heartbeat session. Log as L-022.

## Health-Check False Alarm — CHG-0284 / L-030

- **Incident:** Health-check script reported Anthropic API as degraded (401) when API was actually fine (Yoda was responding on Sonnet).
- **Root cause:** health-check.sh, get-secret.sh, outage-detect.sh, validate-fallback-chain.sh all reading from stale keychain entry (`ainchors-anthropic-api-key`) — different from the key the gateway actually uses in `auth-profiles.json`.
- **Fix:** All 4 scripts updated to read from `auth-profiles.json` first, keychain as fallback.
- **Propagation script updated:** Now also syncs all 3 keychain entries (`ainchors-anthropic-api-key`, `ainchors-anthropic-key-current`, `ainchors-ocl-api-key`) after updating auth-profiles.
- **Source of truth:** `auth-profiles.json` is always authoritative. Keychain = convenience fallback only.
- **L-030:** After any key rotation, run propagation script immediately — it now handles both agent configs AND keychain entries in one pass.

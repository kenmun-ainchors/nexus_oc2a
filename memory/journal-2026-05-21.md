## What Broke

### 1. Tilde-Path Write Failures (CHG-0281 violations — 4 incidents today)
- **What failed:** `scripts/notion-sync-fix.sh`, `linkedin_draft_20260521_v2.txt`, `state/linkedin-queue.json`, `docs/forge-test-verification.md` — all writes used `~/.openclaw/...` instead of absolute paths.
- **Why:** Sub-agents spawned with tilde paths despite CHG-0281 being a NON-NEGOTIABLE rule in RULES.md. Isolated sessions do not expand `~`.
- **Fix needed:** Systemic enforcement, not just a rule. Options: (a) pre-flight path validation in all tools.write calls, (b) `$HOME` env injection in sub-agent spawn, (c) gateway-level path rewrite for tilde prefixes.
- **Prevention:** CHG-0281 needs enforcement, not just documentation. Today's 4 failures are the Nth occurrence. This is now a platform reliability issue, not a discipline issue.

### 2. Forge Workspace Isolation (TKT-0198 — 1 incident today)
- **What failed:** Forge couldn't find 5 JSON state files because it looked in `workspace/forge/` instead of workspace root.
- **Why:** Sub-agent working directory isolation. Forge's cwd was `workspace/forge/` (empty after cleanup). The files live in `workspace/` root.
- **Fix:** TKT-0235 (workspace discipline) proven working for Atlas — needs to be applied to Forge and all other sub-agents.
- **Prevention:** Sub-agent spawn should default to workspace root, not agent-specific subdirectory.

### 3. Alert Fatigue (102 false-positive obs errors)
- **What:** 102 `cron_run_fail` events flooding monitoring from 12 known-interim Ollama drift items (CHG-0363 model assignments).

## What I Learned

1. **Tilde-path bug is now a platform reliability issue.** 4 write failures in one day, across 3 different agents, on a rule that's been NON-NEGOTIABLE for weeks. The rule alone isn't working. Needs code-level enforcement. If a write uses `~`, the tool itself should reject it or rewrite it.

2. **PostgreSQL on OC1 is a milestone, not just a task.** The 5-tier schema landing signals the transition from file-based state to structured data. But the migration (TKT-0198) being blocked by a workspace isolation bug shows the gap between "deployed" and "operational." Infrastructure without migration tooling is infrastructure that isn't being used.

3. **The State Checking Pattern is correct but reactive.** "Check before you act" shouldn't need a non-negotiable rule — it should be the default behavior of the platform. The fact that we had to codify it says something about how agents were operating before today. The next step is making it automatic, not requiring agents to remember.

4. **Angie broke her 7-day silence.** Two Telegram messages to Aria. We don't know what they said yet, but the re-engagement itself is significant. The business stream may be about to restart.

5. **The platform can execute autonomously now.** 6 ticket completions, 10 CHGs, LinkedIn pipeline end-to-end, PostgreSQL deployment, Sources of Truth Register, State Checking Pattern — all in one day with Ken driving only the RTB sprints. The agents are doing real work.
